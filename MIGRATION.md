# MIGRATION.md — Cleaning up the old repo

This package replaces the Week 1 code currently in
`Machine_Translation_For_PSA_Group10`. Copy this package's files into the repo
root, then apply the changes below.

## Delete

- `SRC/collectors.py` — **conflicts with the `SRC/collectors/` package**
  (module-vs-package shadowing bug: imports resolve unpredictably). The
  package replaces it.
- `SRC/preprocessing.py` — empty placeholder; superseded by `SRC/cleaning.py`.
- `SRC/models.py` — empty placeholder; modeling starts in Week 2+.

## Rename

- `SRC/_init_.py` → `SRC/__init__.py` — the current filename is misspelled
  (single underscores), so `SRC` is not a proper package and
  `from SRC.collectors import ...` breaks.

## Replace

- `SRC/sources.py` → replaced by `SRC/collectors/sites.py`
  (12 documented site configs with domains, link patterns, selectors,
  robots/rate-limit-aware fetching).
- `SRC/psa_collector.py` (generic `<p>` grabber, no robots.txt, no rate
  limiting) → replaced by `SRC/collectors/base.py` + `SRC/scraper.py`.
- `SRC/excel_manager.py` → **remove entirely**. It writes a wrong-language
  target column; the tester/translation languages for this project are
  **Kiswahili (sw)** and **Ekegusii (guz)** only. The new schema
  (`SRC/schema.py`) has exactly `Kiswahili` and `Ekegusii` columns — nothing
  else.

## Data

- Old `data/raw/english_psa.csv` rows can be re-added later through the schema
  (`SRC.schema.new_record`) if the team wants to keep them; tag them
  `{"type": "manual", ...}` in Metadata.
- New pipeline output is **`data/processed/psa_parallel_week1.csv`** (plus
  `build_stats.json`), generated by `python scripts/run_week1.py`.

## Notebook

- The old notebook's `from SRC.collectors import collect_redcross` no longer
  exists. Use `from SRC.collectors import collect_site, collect_all` and call
  `collect_site("redcross_news")` or `collect_all()`.
